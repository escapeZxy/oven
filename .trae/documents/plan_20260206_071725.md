# 交互优化方案：引入计划状态管理与历史记录

针对你提出的“增加一栏显示中断和已完成计划”的需求，我设计了以下方案。核心思路是将计划的二元状态（活跃/非活跃）升级为更丰富的三元状态，并在 UI 上提供历史回溯和恢复能力。

## 1. 数据结构升级 (Data Structure)
我们将在 `UserWorkoutPlan` 中引入明确的生命周期状态，而不仅仅是 `isActive`。

- **修改文件**: `packages/core/src/models/log.model.ts`
- **变更**:
  ```typescript
  export interface UserWorkoutPlan {
    // ... existing fields
    
    /** 
     * Plan lifecycle status:
     * - 'active': 正在进行中 (对应 isActive=true)
     * - 'completed': 已跑完所有训练日
     * - 'interrupted': 中途切换了其他计划，或者手动停止
     */
    status: 'active' | 'completed' | 'interrupted';
    
    // isActive 字段保留用于兼容，但逻辑上它等同于 status === 'active'
    isActive: boolean;
  }
  ```

## 2. 核心逻辑适配 (Service Layer)
- **开始新计划 (`startPlan`)**: 将旧的 `active` 计划标记为 `interrupted`，新计划标记为 `active`。
- **完成训练 (`logWorkout`)**: 当检测到是最后一天训练时，将计划状态自动更新为 `completed`。
- **新增功能 (`resumePlan`)**: 允许用户“复活”一个中断的计划。这会将当前正在进行的计划（如果有）标记为 `interrupted`，并将目标计划重新标记为 `active`。

## 3. 界面交互设计 (UI Design)
在首页 (`LogListComponent`) 增加“历史记录”板块。

### 布局结构：
1.  **顶部 (Header)**: 保持不变。
2.  **当前激活 (Active Hero)**: 保持不变。如果没有激活计划，显示引导卡片。
3.  **历史记录 (History) - 新增**:
    - 位于激活计划下方，计划库上方。
    - **Tab 分组**: [ 全部 | 已完成 | 未完成 ]
    - **列表项**:
      - **已完成计划**: 显示金色奖杯图标，展示完成日期。操作按钮：**“再次挑战”** (Restart)。
      - **中断计划**: 显示进度条（例如 "已完成 45%"）。操作按钮：**“继续训练”** (Resume) 和 **“重新开始”**。
4.  **发现计划 (Library)**: 保持不变，作为底部兜底。

## 实施步骤
1.  **Model**: 更新 `UserWorkoutPlan` 接口，添加 `status` 字段。
2.  **Service**: 
    - 更新 `startPlan` 和 `deactivateAll` 逻辑以维护 `status`。
    - 更新 `logWorkout` 处理完成状态。
    - 实现 `resumePlan` 方法。
3.  **Data**: 更新 Seeder 数据，添加一些 `completed` 和 `interrupted` 的模拟数据以便调试。
4.  **UI**: 修改 `LogListComponent`，增加历史记录板块和相应的交互逻辑。
