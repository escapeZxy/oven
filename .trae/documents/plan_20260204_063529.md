# 实现训练会话记录功能 (Workout Session)

我计划将 `WorkoutSessionComponent` 从一个简单的展示页面升级为一个全功能的训练记录表单，其视觉风格将与新设计的首页保持高度一致。

## 设计目标
1.  **沉浸式训练体验**：使用一致的渐变 Hero Header 展示当前训练日信息，提供“完成训练”和“跳过”两个明确的操作入口。
2.  **动态表单记录**：根据当天的训练内容，自动生成一组包含动作、组数、重量和次数的响应式表单。
3.  **交互优化**：为每一个训练动作设计独立的卡片，清晰区分“建议目标”与“实际表现”。
4.  **状态同步**：训练完成后，自动更新后端数据并跳转回首页，首页的进度条将实时反馈这一变化。

## 实施方案

### 1. 数据逻辑扩展 (WorkoutLogService)
- 在 `WorkoutLogService` 中新增 `logWorkout` 和 `skipDay` 方法，封装对核心层 `WorkoutService` 的异步调用。
- 确保调用完成后更新本地的信号量（Signals），触发 UI 的实时刷新。

### 2. 组件逻辑升级 (WorkoutSessionComponent)
- **表单构建**：使用 `FormBuilder` 和 `FormArray` 构建嵌套表单。
    - 每个动作（Exercise）对应一个 `FormGroup`。
    - 动作内部根据组数（Sets）动态生成重量（Weight）和次数（Reps）的输入框。
- **生命周期管理**：在 `effect` 中监听 `todaysWorkout()` 信号，当数据加载完成后自动初始化表单。
- **操作实现**：实现 `onSubmit()` 和 `onSkip()` 方法，处理提交过程中的 Loading 状态和路由跳转。

### 3. UI 界面重新设计
- **Header**：使用与首页一致的 `bg-gradient-to-br from-indigo-600 ...` 渐变背景。
- **动作卡片**：每个动作展示为一个现代化的白底卡片，包含动作名称、建议目标。
- **输入区域**：使用 Material `appearance="outline"` 风格的紧凑输入框，方便用户在手机或平板上快速输入数据。
- **底部操作栏**：固定的操作区域，确保“完成训练”按钮始终触手可及。
